HTML

html은 웹페이지에 표현하는 방식, 문법이라고 생각하면 될 것 같다. 중요한 것은 내가 타이핑한 것이 어떤 의미냐 하는 것.

예를 들어, h1태그를 표현한다고 생각해보자. 이걸 그냥 글자 크고 진하게 만드는 식으로 쓰면 안된다는 것이다. 그렇게 꾸미는 것은 CSS의 역할이다.

파이썬과 다르게 기능들을 외워서 쓸 수 있는게 아니다. 기본적인 기능은 손에 익을테니, 추가로 필요한 기능을 그때그때 구글링하는 식으로 진행하자.

구글링할 때는 mdn의 것을 먼저 보자. firefox 개발자들이 정리해 놓은건데, 거의 업계표준같은 느낌.

CSS

html에 쭉 적혀있는 코드를 예쁘게 만드는 방법인 것 같다.

html과 마찬가지로 외워서 쓸 수 있는 것이 아닌 것 같으니, 구글링하자.

css의 c는 Cascade인데, 스타일이 적용되는 순서를 설명해준다(명시도). 같은 우선순위일 경우 가장 나중에 선언한 것이 적용된다. (태그의 클래스 선언은 순서 상관 없음)

복잡하게 그 순서 외우지 말고, 99%는 클래스 선언해서 스타일 지정하는 식으로 진행하자.
